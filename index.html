DSTNY

A group project of the General Assembly coursework collaborated with Andrea Lau , Ho Jia Hern & Siu Sing Lai. This Social Polling Application is created with Ruby on Rails & MongoDB to let users help people make decisions at forks in their lives. (Aug 2020)

Planning Process

Wireframe, ERD & Trello:

Previous Next

Models:

  • Users
  • Questions (has_many_Comments/Options)
  • Options (belongs_to_Questions)
  • Comments (belongs_to_Questions)
  • Categories(#Tag)
Outcome

Features:

  • Cloudinary
  • Avatars
  • User & Admin Login
Challenges / Learnings
  • Pull requests were too large and required entire team effort to resolve due to overlapping conflicts, future teamwork should commit smaller and more frequent changes and ensure better segregation of tasks
  • Link_to, overwrote clickable container
  • Models were planned well
  • Conflict resolution was done carefully, PR were reviewed and tested by another dev whom was not involved in writing that code
  • Possibly creating more join-tables vs 1-to-many relation
  • Spend some time on leaving comments/clean up indentations to facilitate teams’ reading when alteration needed
Possible Future Additions
  • User to user chat
  • Deadline for polls
  • OP able to choose incentivization award to majority

Cheapo Beepo

Backend project using MongoDB to scrape websites like NTUC & Cold Storage, then populate prices & items for users to compare. (Jul 2020)

Planning Process

Wireframe:

Previous Next

Models:

  • Shop (has_many_Items)
  • Items (belongs_to_Shop)
  • Users
Outcome

Features:

  • Search Function
  • Webscraping
Challenges / Learnings
  • NTUC site was messy (non standardized naming convention for div classes) and scraping it resulted in huge time loss, then only reaping 30% results due to lazy loading and REACT
  • Different Units and UOM made comparison more complicated
Possible Future Additions
  • Comparison feature
  • Geolocation for shops
  • Filters for price & distance
  • Ratings system

Pattern Matching Game

First project ever made to familiarize CSS, HTML and JS. Similar to simon says, click the flashing squares in the correct sequence. (Jun 2020)

Features:

  • Randomized colors and patterns
  • Squares from prev rounds maintains own color
  • Increase squares in squared numbers
Challenges / Learnings
  • Incorrect interval resetting caused overlapping timer after each round
  • Random flashing glitch caused by chrome alerts, fixed by removing alerts and use modals instead
Possible Future Additions
  • Stopwatch
  • Local Storage Highscores
