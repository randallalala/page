DSTNY

A group project of the General Assembly coursework collaborated with Andrea Lau & Siu Sing Lai. This Social Polling Application is created with Ruby on Rails & MongoDB to let users help people make decisions at forks in their lives. (Aug 2020)

Planning Process

Wireframe, ERD & Trello:

Previous Next

Models:

  • Users
  • Questions (has_many_Comments/Options)
  • Options (belongs_to_Questions)
  • Comments (belongs_to_Questions)
  • Categories(#Tag)

Outcome

Features:

  • Cloudinary
  • Avatars
  • User Login
Previous Next

Challenges / Learnings
  • Pull requests were too large and required entire team effort to resolve due to overlapping conflicts, future teamwork should commit smaller and more frequent changes and ensure better segregation of tasks
  • Link_to, overwrote clickable container
  • Models were planned well
  • Conflict resolution was done carefully, PR were reviewed and tested by another dev whom was not involved in writing that code
  • Possibly creating more join-tables vs 1-to-many relation
  • Spend some time on leaving comments/clean up indentations to facilitate teamsâ€™ reading when alteration needed
Possible Future Additions
  • User to user chat
  • Deadline for polls
  • OP able to choose incentivization award to majority
